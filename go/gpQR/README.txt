■動作
本プログラムは、指定されたテキストまたはテキストファイルから、任意のサイズ・DPI・余白を設定してフッター文字付きのQRコード（PNG画像）を生成するコマンドラインツールです。

■ 主な機能と特徴
1. **テキストファイル入力と文字コード自動判別**
   * `--textfile` オプションでテキストファイルを指定可能。
   * ファイルの文字コードが `UTF-8` または `Shift_JIS` であるかを自動で判別して取り込みます。
2. **解像度（DPI）および余白（マージン）の指定**
   * `--dpi` で印刷や画面表示に合わせた解像度（300/600dpiなど）を指定できます。
   * `--margin` でQRコードの周囲に指定ピクセル分の白い余白を作ることができます。
3. **出力ファイル名の自動生成**
   * `--o` に既存のディレクトリ、またはディレクトリパス（末尾が `/` や `\`）を指定した場合、`qr_年月日_時分秒.png` というファイル名で自動保存されます。
4. **親切なエラー通知**
   * 文字数が多すぎてQRコードの容量（上限）を超えてしまった場合、プログラムが強制終了（panic）せず、分かりやすいエラーメッセージを表示します。

■オプション一覧（コマンドライン引数）

 オプション  | 必須/任意 | 説明 | デフォルト値 |
 --qrtxt     | ※        | QRコードにする文字列を直接指定します。 | `""` |
 --textfile  | ※        | QRコードにするテキストファイルのパスを指定します。 | `""` |
 --o         | 任意      | 出力先ファイルパス、または保存先ディレクトリを指定します。 | `"gpQR.png"` |
 --size-mm`  | 任意      | QRコード本体のサイズ（ミリメートル単位）を指定します。 | `0` (40.0mm相当) |
 --auto-size | 任意      | 文字数に応じてQRコードのサイズを自動調整します（30〜50mm）。 | `false` |
 --ftext     | 任意      | QRコードの下部に印刷するフッターテキスト（ラベル等）を指定します。| `""` |
 --level     | 任意      | QR誤り訂正 (L,M,Q,H) デフォルトM
 --dpi       | 任意      | 解像度（DPI）を指定します。（例: 300, 600） | `300` |
 --margin    | 任意      | QRコードおよびテキストの周囲に設ける余白の幅（ピクセル単位）。 | `40` |

※注意**: `--qrtxt` または `--textfile` の**いずれか一方は必ず指定**する必要があります。両方指定された場合は `--textfile` が優先されます。

■具体的な使い方・コマンド例

### ① 基本的な使い方（テキスト直接指定）
「Hello QR」という文字のQRコードを、デフォルト設定（300dpi、サイズ40mm、余白0、ファイル名 `gpQR.png`）で生成します。
go run gpQR.go --qrtxt "Hello QR"
gpQR.exe --qrtxt "Hello QR"
```

### ② テキストファイルから生成（Shift_JIS / UTF-8 自動判別）
`input.txt`（UTF-8でもShift_JISでも可）に記述された長文からQRコードを生成します。
go run gpQR.go --textfile "./input.txt"
gpQR.exe --textfile "./input.txt"

### ③ 高解像度（600dpi）＋余白（20px）＋フッター文字を追加
印刷用に解像度を高め、周囲に20ピクセルの余白を取り、下部に管理番号を表示します。
go run gpQR.go --qrtxt "https://example.com/item123" --dpi 600 --margin 20 --ftext "管理番号: 123" --o "printed_qr.png"
gpQR.exe --qrtxt "https://example.com/item123" --dpi 600 --margin 20 --ftext "管理番号: 123" --o "printed_qr.png"

### ④ 保存先ディレクトリだけを指定（ファイル名自動生成）
既存の `C:\qr_outputs` というディレクトリ（またはマージ先のフォルダ）に出力したい場合、ファイル名を省略できます。
go run gpQR.go --qrtxt "ディレクトリ指定テスト" --o "C:\qr_outputs\"
gpQR.exe --qrtxt "ディレクトリ指定テスト" --o "C:\qr_outputs\"
```
* **生成されるファイル名の例**: `C:\qr_outputs\qr_20260703_150000.png`

### ⑤ 文字数に応じたサイズ自動判定
文字数が多い場合に自動でサイズを大きめ（最大50mm）に調整します。
go run gpQR.go --textfile "long_text.txt" --auto-size
gpQR.exe --textfile "long_text.txt" --auto-size

■エラー時の挙動について
### QRコードの容量オーバー（文字数制限）が発生した場合
QRコード（特に誤り訂正レベル `Low`）には格納できるデータ量に上限があります。上限を超えるような大量のテキストを入力した場合、画面に以下のようなエラーが表示され、安全に処理を中断します。

【エラー】QRコードの生成に失敗しました:
QRコードの容量オーバーです。文字数を減らすか、エンコード設定を見進してください。(詳細: text too long to encode)

**対処法:**
* 埋め込むテキストの文字数（バイト数）を減らしてください。
* URLなどの場合は、短縮URLサービスなどを利用して文字列を短くしてください。
